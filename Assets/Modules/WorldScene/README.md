# WorldScene

WorldScene is a lightweight scene orchestration module for loading a persistent world scene and additive sub-level scenes.

## Runtime

- Assembly: `WorldScene.Runtime`
- Root namespace: `WorldSceneModule.Runtime`
- Runtime dependencies: UnityEngine and UniTask.
- Runtime code must not reference `UnityEditor`, `AssetDatabase`, or editor scene loading APIs.
- `LevelRegistry` stores editor object references for authoring, but runtime loading uses cached scene paths generated by the editor sync tool.

Main flow:

```csharp
using System.Threading;
using WorldSceneModule.Runtime;

await driver.InitializeWorldAsync(CancellationToken.None);
WorldSceneLoadResult result = await driver.LoadLevelAsync("Level1", CancellationToken.None);
```

Dispose or cancel caller-owned `CancellationTokenSource` instances outside this module.

## Editor

- Assembly: `WorldScene.Editor`
- Editor tools live under `Editor/`.
- `LevelRegistryEditor.SyncRegistry` builds a deterministic sync plan, updates cached paths, and creates per-level `LevelModuleData` assets.
- The Inspector sync button previews scene and asset creation counts before writing changes.
- Sync also warns when registered scene paths are not enabled in Build Settings.
- Missing registered scenes can be appended explicitly from `Tools/World Scene/Add Missing Scenes To Build Settings`.
- Inspector UI uses UI Toolkit assets under `Editor/Styles`.

## Tests

- Test assembly: `WorldScene.Editor.Tests`
- Command line:

```bash
Tools/CI/run-worldscene-editmode-tests.sh
```

The command writes:

- `Logs/TestResults/worldscene-editmode-results.xml`

## Rules

- Keep runtime scene loading based on cached paths.
- Keep editor asset discovery in Editor assemblies.
- Keep level module data generic; concrete gameplay systems should register their own ScriptableObject data without adding compile dependencies here.

Remaining gaps are tracked in `INDUSTRIALIZATION_GAPS.md`.
